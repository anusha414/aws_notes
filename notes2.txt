AWS ECS(ELASTIC CONTAINER SERVICE):-(Video:- https://www.youtube.com/watch?v=nnIFkEOzNp4)

1. ECS is a container orchestration enviornment that is not dependent on k8s.
2. As k8s is open source, openshift eks and many other coe use k8s as base.
3. ECS does not use pods,deployments or services, they have their own resources like tasks, services.
4. ECS is restricted only to AWS and it is not open source.
5. Disadvantage:- if you want to move to Azure from ECS, you will have to again create pods, deployments, services and all, as ECS resources cannot be reused.
6. ECS does not have all features that k8s supports. eg:- custom resource, argocd, fluxcd
7. ECS can be used with EC2 or Fargate.

Structure of ECS:-
1. You will create a ECS cluster, use either fargate or create EC2 instances to deploy containers.
2. In k8s we used to create pod yaml to deploy containers, In ECS we will use task definition to deploy the containers.
3. You can create services to add load balancing capabilities.
